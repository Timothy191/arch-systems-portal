- One pytest package (`tests/`) mirrors the top-level `memex` source tree: each major subsystem has its own `test_*.py` file (e.g. `test_cluster_runner.py`, `test_memory_tool.py`, `test_mcp_server.py`, `test_cli.py`).
- `conftest.py` is the single global fixture: it pins `MEMEX_REGISTRY_PATH` to a temp file and provides an autouse session-scoped fixture that rewrites `memex.watcher.registry.REGISTRY_PATH` so no test touches the user's live registry.
- Tests are split by isolation level: unit tests use in-process mocks / `unittest.mock.AsyncMock` / local `_StubGraphClient` classes; integration tests that need a live Neo4j are tagged `@pytest.mark.integration` (e.g. `test_mcp_queries_integration.py`, `test_bidirectional.py`) and excluded from the default baseline run.
- Async code is exercised via `@pytest.mark.asyncio`; synchronous CLI entry points are driven by patching `sys.argv` and calling `cli.main()` directly rather than invoking a subprocess.
- External dependencies (Neo4j driver, Anthropic SDK) are replaced by small in-memory fakes defined inside the test files themselves (`_FakeDriver`/`_FakeClient` in `test_memory_tool.py`, `_StubGraphClient` in `test_cluster_runner.py`), keeping the suite runnable without any external services.
