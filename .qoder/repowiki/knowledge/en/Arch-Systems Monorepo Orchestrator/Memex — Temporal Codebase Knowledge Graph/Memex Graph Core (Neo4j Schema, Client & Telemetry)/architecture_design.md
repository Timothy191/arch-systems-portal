All children in this package depend on three cross-cutting contracts defined here:
- `schema.py` declares every persistent node type (`Symbol`, `Module`, `Decision`, `Cluster`, `ClusterSummary`, …) plus the v0.3.0 cross-node fields (`write_policy`, `access_count`, `last_reinforced_at`) and the `WRITE_POLICIES` / `check_write_policy` ACL gate that `cluster_runner` enforces when persisting Cluster nodes.
- `client.py` exposes a process-wide `GraphClient.get_instance()` singleton backed by Graphiti + Gemini; both `cluster_runner` (Leiden grouping) and `confidence_decay_stats` (nightly decay/sweep) acquire it via `get_graph_client()` rather than constructing their own driver.
- `telemetry.py` + `otel.py` form a dual-output metrics pipeline: `record_tool_call` writes per-call aggregates into `~/.memex/telemetry.db` (SQLite, WAL) while `otel.record_token_metrics` / `set_decision_confidence` emit the same data as OpenTelemetry counters/gauges under the `gen_ai.*` semantic conventions; children call these helpers after each tool invocation so token savings and decision confidence appear in both stores.

Children are otherwise isolated: `cluster_runner` owns Leiden/HDBSCAN orchestration, `confidence_decay_stats` owns the nightly scheduler and decay math, but they never import each other — they converge only on the schema models, the Graphiti client, and the telemetry hooks.
