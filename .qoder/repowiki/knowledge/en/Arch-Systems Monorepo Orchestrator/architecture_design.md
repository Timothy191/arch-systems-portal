The root is the integration layer for a polyglot monorepo: `pnpm-workspace.yaml` declares `apps/*` and `packages/*` as the JS/TS workspace (with a shared `catalog:` pinning React, Tailwind, Supabase, Zod, etc.), while Python tooling lives under `tools_*` and is managed separately via Poetry. `nx.json` defines cross-project targets (`build`, `lint`, `type-check`, `test`) with `targetDefaults.dependsOn: ['^build', '^codegen']` so apps automatically depend on `@repo/ui` and the Kysely-generated types from `packages/supabase`. `eslint.boundaries.cjs` consumes generated dep-constraints produced by `tools_policy`'s policy compiler to enforce scope boundaries across all children. Docker Compose files compose runtime services — `docker-compose.portal.yml` builds the Next.js portal behind nginx; `docker-compose.production.yml` overlays production overrides for n8n, Flowise, Redis, and ClickHouse; `docker-compose.monitoring.yml`, `.security.yml`, and `.tools.yml` add observability and dev-tool stacks. `ci/` GitHub Actions workflows drive cache warmup, policy gates, and rollback against the same compose/Nx surface. Root scripts (`deploy.sh`, `dev.sh`, `quality`, `policy:gen`, `ensure-reachability`) are the single entry points that invoke child projects through Nx/pnpm filters rather than each child being bootstrapped independently.
