Scripts expect to be invoked from the repository root as `./scripts/<script>`; many require `sudo` for port clearing or systemd control. `deploy.sh` accepts `local|staging|production` plus flags like `--dry-run --skip-build --migrate-only --rollback --force --lightweight --no-browser`. `dev.sh` supports `--quick|-q --tools|-t --cms --overview --all --force|-f`. `preflight-checklist.sh --fix` attempts idempotent repairs before deployment.
