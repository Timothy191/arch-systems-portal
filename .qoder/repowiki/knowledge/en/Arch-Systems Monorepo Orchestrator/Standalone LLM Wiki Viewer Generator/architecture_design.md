Two-file pipeline: `generate.js` is the sole generator — it walks `/home/timothy/Project/Arch-Mk2/wiki`, parses YAML frontmatter via a hand-rolled parser, converts Markdown to HTML (headings, tables, lists, blockquotes, code blocks, bold/italic, external links) while resolving `[[slug]]` wikilinks against an in-memory slug index, then concatenates all pages, CSS, and client-side JS into one `viewer.html`. The generated HTML embeds the page data as JSON and runs a browser-side router (`#page=<slug>`) plus features: sidebar grouped by page type (index/schema/log/entity/concept/comparison/query/raw/other), Ctrl+K global search over titles/tags/content, an SVG force-directed link graph seeded from the current page's neighbors, and a static system architecture diagram. `view.sh` is a thin launcher that re-runs the generator and opens the output via `xdg-open` / `open` / Python webbrowser. Dependency direction is strictly one-way: script → filesystem → single HTML artifact; the viewer has no runtime dependencies on the rest of the repo.
