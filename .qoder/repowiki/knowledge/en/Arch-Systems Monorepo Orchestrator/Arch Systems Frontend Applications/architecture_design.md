The `apps/` directory is a flat collection of sibling Next.js apps with no shared runtime code between them; cross-app wiring is external to this scope (handled by the monorepo's build/deploy layer). Each child owns its own `next.config.mjs`, `package.json`, and `project.json`, so they are compiled, versioned, and served as independent deployments. The only structural commonality is that all three follow the Next.js App Router layout (`app/layout.tsx`, `app/page.tsx`) and ship their own Tailwind/PostCSS configs rather than sharing a base.
