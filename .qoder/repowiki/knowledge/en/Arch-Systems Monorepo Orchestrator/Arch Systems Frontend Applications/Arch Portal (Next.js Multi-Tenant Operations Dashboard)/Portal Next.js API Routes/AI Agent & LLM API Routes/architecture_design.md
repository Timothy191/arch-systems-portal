Each subdirectory under `apps/portal/app/api/ai/<feature>/route.ts` is an independent Next.js Route Handler (exported `POST`) implementing one AI feature. All routes share the same request pipeline: authenticate via Supabase (`createServerSupabaseClient().auth.getUser()`), validate body with Zod schemas from `@/lib/api/schemas`, enforce rate limits and body size through middleware wrappers (`withRateLimit`, `withBodyLimit`), call an LLM via the shared `chat` provider (`@/lib/ai/providers` using `DEFAULT_MODEL`), then return JSON wrapped by `applyCors`. The `/chat` route is the only stateful endpoint — it builds an initial LangGraph-style agent state with `createInitialAgentState`, runs `runAgentGraph`, and defers memory persistence to either `globalThis.waitUntil(finalizeAgentGraph(...))` or an Inngest `ai/memory-persist` event as a fallback. Structured-output routes (`/predict`, `/safety`) pass a `SYSTEM_PROMPT` instructing the model to emit strict JSON, then re-parse the raw string against a schema in `@/lib/ai/schemas` and fall back to a safe default object when parsing fails. Error paths are centralized through `logError` with a per-route context tag.
