Organized as a Next.js Route Group `(hub)` so its URL prefix is stripped at build time. `layout.tsx` is the route-group shell: it authenticates via `getUserSafely`, fetches the user's accessible departments through a read-replica Supabase client, caches the department list with React `cache()`, and wraps all children in a `<BottomNav>`-fed container. `page.tsx` is the main hub landing — an async server component that uses `dynamic = 'force-dynamic'` and streams slow data (production trend) inside a nested `Suspense` boundary (`ProductionTrendSection`) while fast counts/alerts/department cards render immediately. Data access follows a consistent pattern: each query helper (`getDashboardCounts`, `getRecentAlertEvents`, `getEmployeeDepartments`, `getProductionTrendData`) is wrapped by `cachedRSC` + `withCache` using `@repo/redis` `CacheCategory.METRICS`/`AUTH`, keyed on `[userId, today]` and tagged with `auth:${userId}` plus `table:<name>` tags for invalidation; queries go through `createReadReplicaClient(cookieList)` to propagate session cookies. `executive/page.tsx` is an admin-only KPI page gated by an `employees.role` check (`admin | manager`), computing MTD production/fuel/hours/breakdowns from parallel Supabase reads and exposing CSV/PDF export helpers. `error.tsx` is a client-side error boundary that classifies errors via `isAppError`/`isNotFoundError` and logs through `logError`; `loading.tsx` provides a `GlassSkeleton` placeholder matching the hub's card grid shape.
