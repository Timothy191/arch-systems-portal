- app/admin/page.tsx is the server-side route entry: it authenticates via createServerSupabaseClient, enforces employees.role === 'admin' (redirecting non-admins to /), resolves the active tab from searchParams.tab against a local TABS whitelist, then renders AdminTabsClient with each tab component as conditional children inside a Suspense boundary.
- features/admin/components/AdminTabs.tsx is a pure client UI wrapper around @repo/ui Tabs; AdminTabsClient.tsx bridges Next.js router (useSearchParams + router.replace) into the tab value, keeping URL state in sync without scroll reflow.
- Each domain lives under features/admin/tabs/<Name>Tab.tsx as a self-contained "use client" React component owning its own Supabase client (createBrowserSupabaseClient), local state, filtering/search, and dialog forms — no cross-tab shared state.
- Mutations are isolated to features/admin/actions/*.ts files marked "use server"; they centralise admin authorization via an assertAdmin() helper that re-checks employees.role per call, then perform Supabase writes followed by cache invalidation through cacheInvalidateTags("table:*") plus revalidateTag/revalidatePath("/admin").
- Dependency direction is strictly one-way: tabs -> actions; components depend only on @repo/ui primitives and shared @repo/supabase clients; nothing in this module imports sibling tabs or actions outside their own file.
