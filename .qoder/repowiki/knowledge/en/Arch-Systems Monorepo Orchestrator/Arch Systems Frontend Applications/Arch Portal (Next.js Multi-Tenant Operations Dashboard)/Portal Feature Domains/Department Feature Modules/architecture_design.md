Each child under `apps/portal/features/departments/` is an independent department feature (control_room, engineering/breakdowns, safety, satellite, tools) that follows the same contract: it ships a client-only React page plus co-located components, actions, and types, and is wired into the portal by importing its page component from the parent `features/departments` index so the router can mount it under `/departments/<feature>`. There is no cross-feature runtime communication — features are isolated bundles loaded on demand, and they share only the portal's top-level layout, auth/session context, and shared UI primitives consumed via relative imports from sibling feature packages.
