Each top-level directory under `features/` is an independent feature domain (admin, analytics, departments, hub, webhooks, access-control) containing its own React components and, where needed, server actions (`actions/`) — there is no shared feature library; features compose via direct imports from sibling directories. Cross-feature sharing is limited to the shared `@repo/ui/GlassCard` primitive and CSS variables (`--text-heading`, `--bg-primary`, etc.) consumed uniformly by every component. Heavy third-party libraries (UniverSheet, Fuxa iframe, charting) are loaded lazily through `next/dynamic` with `ssr: false` so they never reach the server bundle. The only inter-feature coupling visible here is the hub's `DepartmentCard` referencing department names and the tools page polling `/api/tools/status` for a unified tool registry.
