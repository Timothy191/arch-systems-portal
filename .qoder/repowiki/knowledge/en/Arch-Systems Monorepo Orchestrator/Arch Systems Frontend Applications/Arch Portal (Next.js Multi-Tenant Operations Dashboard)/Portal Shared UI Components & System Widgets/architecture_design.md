Flat `components/` tree organized by feature domain — `system/` (tray, ticker, split-window layout, viewport helpers), `nav/` (bottom nav, services dropdown), `weather/`, `ai/`, `monitoring/`, `clock/`, `ui/` (atomic primitives like PrecisionInput). Each file is a self-contained client component (`"use client"`) that owns its own state via local hooks or custom hooks from `@/hooks/*`. Cross-cutting concerns are isolated in sibling files: `FocusModeProvider.tsx`, `SmoothScrollProvider.tsx`, `RouteAnnouncer.tsx`, `PerformanceListener.tsx`, `WebVitalsReporter.tsx`, `OfflineBanner.tsx`, `CommandBar.tsx`, `HeaderWidgets.tsx`, `BottomWidgetBar.tsx`, `ViewportBoundaries.tsx`. Styling is exclusively Tailwind with CSS variables (`--accent-*`, `--text-*`, `--bg-*`) consumed through `cn()` from `@repo/ui/lib/utils`; icons come from `lucide-react`. External UI primitives (`Popover`, `GlassCard`) are imported from `@radix-ui/react-popover` and `@repo/ui/GlassCard`. Data fetching is done inline with `fetch` (e.g. `/api/health`, `@/lib/weather-api`) rather than a shared data layer, keeping components read-only consumers of their own subscriptions.
