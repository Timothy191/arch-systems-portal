Built as a Next.js App Router segment under `app/(departments)/engineering/`. The root `layout.tsx` composes shared chrome via `DepartmentLayout`, pins the active department through `ActiveDepartmentSetter`, injects an `AIAssistantWrapper` scoped to "Engineering Department", and resolves its tab list from `getDepartmentTabs("engineering")` against the central `DEPARTMENTS` registry. The root `page.tsx` is a server component (`dynamic = "force-dynamic"`) that fetches breakdown metrics (active count, resolved today, recent items) in parallel from a Supabase read replica via `createReadReplicaClient()` and renders them inside `GlassCard` tiles linking to `/engineering/breakdowns` and `/engineering/tire-management`. The `tire-management/` sub-route mirrors this pattern: a minimal layout passthrough and a server-component page that calls `getDepartmentContext({ department: "engineering" })` for auth scoping and currently renders a stub UI pending database tables. Dependency direction is one-way outward — this module depends on `@repo/ui`, `~/lib/departments`, `~/lib/dept-context`, `@/components/nav`, and `@/components/ai`, but exposes no internal APIs.
