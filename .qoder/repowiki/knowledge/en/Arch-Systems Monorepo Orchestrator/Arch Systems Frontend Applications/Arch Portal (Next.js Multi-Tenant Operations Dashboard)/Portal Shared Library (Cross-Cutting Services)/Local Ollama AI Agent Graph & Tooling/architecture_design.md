The module is organized as a layered agent runtime:
- `agent-graph.ts` defines the explicit finite-state machine (authenticate → rateLimit → resolveContext → loadMemory → gatherContext → executeTools → callLLM → output → saveMemory) with each node returning a partial `AgentState`; `runAgentGraph` walks the `NODE_MAP` until `nextNode === 'END'`, wrapping every node in `withSpan('agent.node.<name>')` for tracing.
- `agent-state.ts` holds the shared mutable state shape plus `reduceState` so nodes never mutate directly.
- `tool-dispatch.ts` replaces regex keyword matching: it first tries Ollama's native `/api/chat` tool-calling API, then falls back to parsing a JSON `{tool, args, confidence, reason}` block from free-form text; confidence ≤ 2 routes to a clarifying prompt instead of firing a tool.
- `tools.ts` declares four Supabase-backed tools (`machineStatus`, `fleetStatus`, `shiftLogs`, `delays`) as `{ description, inputSchema: z.object(...), execute }` objects collected into `aiTools`; `executeToolsNode` calls them via `aiTools[name].execute(args)` and caches results per-tool with TTLs defined in `TOOL_CACHE_TTL`.
- `embeddings/index.ts` exposes `generateEmbedding` / `batchGenerateEmbeddings` behind a three-tier cache: L1 in-process Map keyed by `(hash, userId)`, L2 Postgres table, then the primary provider — `provider.ts` currently instantiates a single `OllamaEmbeddingProvider` backed by `ollamaEmbed` with model `nomic-embed-text:latest` and fixed `EMBEDDING_DIMENSIONS = 768`.
- `providers.ts` re-exports thin wrappers around `ollamaChat` / `ollamaChatStream` / `ollamaEmbed` that log errors via `logError` with a `context` tag.
- Cross-cutting concerns live in sibling files: `rate-limiter.ts` (global + per-tool rate limiting), `tool-cache.ts` (in-memory LRU with per-tool TTL), `memory.ts` (hybrid semantic+episodic store using embeddings), `prompts.ts` (system prompts including `toolDispatch`), `cost-tracker.ts`, `chunking.ts`, and `schemas.ts`.
- All external I/O goes through `@repo/supabase/server` (`createServerSupabaseClient`) and `@repo/supabase` (`withSpan`); the only network dependency outside Supabase is direct `fetch()` against `${OLLAMA_URL}/api/chat`.
