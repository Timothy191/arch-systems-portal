- `packages/database` is a pure-SQL package (`@repo/database`) exporting its `migrations/*.sql` files via `exports`; it has no runtime dependencies and is consumed by other packages only as source-of-truth migrations. It ships local dev/test scripts that drive `pnpx supabase start`, `db push/reset`, and SQL-based security tests against a local Supabase instance.
- `packages/ui` (`@repo/ui`) is a React 19 + Tailwind v3 component library. All styling is delegated to `@repo/theme/tailwind.config.ts` (re-exported from `tailwind.config.ts`). Components are split into three layers: base primitives under `src/components/ui/` (Radix-based), layout/page chrome under `src/components/*.tsx`, and workflow-builder nodes/edges under `nodes/` and `edges/`. Animation helpers live in `src/components/motion/` (animejs) and `motion-primitives/` (framer-motion). The package exposes every public component through explicit `package.json` `exports` entries so consumers import by stable name rather than path.
- `packages/supabase` (`@repo/supabase`) is the single application boundary over Supabase. `config.toml` pins Postgres 15 with a transaction-mode pooler and enables auth/storage/realtime/edge-runtime/analytics for local dev. Runtime clients are split by execution context: `client.ts` creates a browser client using cookie-based session persistence; `middleware.ts` builds an SSR client bound to Next.js `NextRequest` cookies with HttpOnly/Secure/Lax defaults; `server.ts` provides a Server Component client plus `getUserSafely` that swallows refresh-token errors. A separate `kysely.ts` module opens a raw `pg.Pool` against `DATABASE_URL` for complex queries, with a hand-maintained `KyselyDatabase` interface mirroring key tables. `database.types.ts` is generated from the shared migrations and re-exported alongside these modules via `exports`.
