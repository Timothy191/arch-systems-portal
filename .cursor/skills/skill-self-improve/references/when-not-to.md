# When not to self-improve

- **Policy** — never invent, soften, or fork `AGENTS.md` / §18 never-dos
- **Duplicates** — if a skill already covers the procedure, patch that skill; do not create a twin
- **Secrets** — never write keys, tokens, or `.env` contents into skills
- **Product packages** — do not put agent procedures under `apps/` or `packages/`
- **One-off fluff** — do not distill trivia that will not recur
- **Heavy evolution** — do not add DSPy/GEPA dependencies to the monorepo; use offline Hermes self-evolution outside if needed
- **Replace review** — distill does not replace `sceptic` or formal Alignment Score
