Four cooperating files form a layered pipeline:

- `cluster.py` is the pure algorithm core: builds a weighted `networkx.Graph` from directory co-location (weight 1), module imports (weight 2) and symbol-call aggregation (`log(1+calls)`), runs `graspologic.partition.hierarchical_leiden`, names clusters by TF-IDF over Decision texts (falling back to parent-dir prefix), pins names across re-runs via Jaccard similarity against prior Cluster nodes, then applies user overrides. All helpers are synchronous and testable without Neo4j; only `run_cluster_pass` is async and optionally takes a graph client.
- `cluster_overrides.py` loads `.memex/clusters.yaml` (flat or nested `{clusters: {name: [paths]}}`) into a `{module_path: cluster_name}` map; errors are logged and never raised so a malformed file cannot crash the watcher.
- `cluster_runner.py` is the CLI orchestrator exposing two entry points: `run_init_cluster_pass` (filesystem-only discovery for `memex init`, best-effort persistence) and `run_cluster_command` (Neo4j-backed `memex cluster` supporting `--dry-run`, `--rerun`, `--refresh-summaries`). Persistence goes through `write_cluster_assignments`, which enforces ACL via `check_write_policy('Cluster', 'cluster')`, upserts Cluster nodes with `add_episode` + Cypher MERGE, writes CONTAINS edges, tombstones stale edges/nodes, and sets `last_reinforced_at` on each reinforcement pass.
- `cluster_summary.py` calls Gemini Flash (`gemini-2.5-flash`) to synthesize a one-line purpose summary per cluster and writes it back to the graph; also exposes `_embed_text` for backward compatibility.

Dependency direction is strictly inward: runner → cluster (+ overrides + summary); cluster → overrides; nothing inside this module is imported by the rest of memex except the public `run_cluster_pass` / `ClusterAssignment` surface.
