Three cooperating modules with a clear dependency direction:

- `confidence.py` is the pure-function core: it defines the TempValid three-regime exponential-decay formula (validated floor at 0.7, unvalidated cap at 0.5, staleness threshold 0.3) as constants (`LAMBDA_*`, thresholds) plus `current_confidence`, `is_stale`, and `is_cold`. It accepts either Cypher dict rows or Pydantic/attribute-style objects via `_get`, and normalises Neo4j `DateTime`/ISO strings through `_as_datetime`.
- `decay.py` owns the `DecayScheduler` class built on `apscheduler.schedulers.asyncio.AsyncIOScheduler`. Its `decay_task` mirrors the Python formula in an inline Cypher query (`_STALE_REFRESH_QUERY`) to refresh the cached `stale` boolean across all `Entity` nodes in one round-trip, then delegates tombstoning of cold nodes to `memex.graph.archive.tombstone_cold_nodes` per active repo from `memex.watcher.registry`. The class name is kept for daemon wiring compatibility even though it no longer mutates confidence.
- `stats.py` is a read-only aggregation layer: `_query_db_sync` runs four time-bounded SQLite queries against `TelemetryDB.tool_calls` (today / 7d / 30d / lifetime), computes top tools and agent-client breakdowns (with a hardcoded display-name map), while `get_stats_data` additionally pulls validation health counts from Neo4j via `get_graph_client`. `print_rich_stats` renders the result via `rich` for the CLI; the async entry point uses `asyncio.to_thread` to keep the blocking SQLite work off the event loop.

No internal sub-packages — the boundary is file-level, with `decay` importing constants from `confidence` and both `decay` and `stats` reaching out to `memex.graph.client` for Neo4j access.
