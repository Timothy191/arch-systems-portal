Single-entry TypeScript extension (`src/extension.ts`) wired via `package.json` contributions: a status bar item, five commands (`memex.openGraph`, `memex.refreshGraph`, `memex.showFileContext`, `memex.startServer`, `memex.stopServer`), and a `WebviewViewProvider` registered under `memex-graph`. The extension is split into four cooperating classes:

- `MemexClient` (src/memexClient.ts) — pure HTTP client over Node's built-in `http` module; calls `/health`, `/graph`, `/stats?days=1` and maintains one long-lived SSE connection to `/events` with manual line-buffering for `event: graph_updated`.
- `MemexGraphPanel` (src/graphPanel.ts) — implements `vscode.WebviewViewProvider`; owns the webview lifecycle, reads `webview/graph.{html,css,js}` from disk, rewrites relative asset paths via `asWebviewUri`, and bridges messages between host and webview (`updateGraph`, `highlightActiveFile`, `openFile`).
- `MemexStatusBarItem` (src/statusBar.ts) — thin wrapper around a right-aligned status bar entry showing offline / warning / connected state plus tokens-saved count.
- `extension.ts` orchestrates all three, spawning the memex server as a child process via `uv run memex serve --repo <workspace> --transport http --port <config.serverPort>` and running a 5s health-check loop plus a 15-min stats poll.
  The webview layer (`webview/graph.js`) is plain browser JS loaded from the extension bundle; it uses D3 v7 (forceSimulation + zoom + drag) and communicates with the host through `acquireVsCodeApi().postMessage`. Build is driven by a custom `esbuild.js` bundler targeting CJS node16 with `vscode` externalized, producing `dist/extension.js` consumed by `package.json.main`.
