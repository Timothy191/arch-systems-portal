Single-package leaf module exposing a flat public surface via `__init__.py` that re-exports three subsystems:

- `settings.py`: a single `Settings(BaseSettings)` class built on `pydantic_settings`. It declares every application setting as a typed `Field`, loads from `.env` at the project root (`Path(__file__).resolve().parent.parent / ".env"`) with case-insensitive env vars and unknown-var ignore, and augments validation with `@field_validator` methods plus helper predicates (`is_production`, `get_neo4j_config`).
- `feature_flags.py`: an enum-driven flag registry. `FeatureFlag` enumerates all flags; `FeatureFlagConfig(BaseModel)` carries per-flag metadata (enabled, rollout percentage, allowed environments); `FeatureFlagManager` holds defaults in `_initialize_defaults()` and evaluates them against the current environment. A module-level `_feature_flag_manager` singleton is lazily created by `get_feature_flag_manager()`, which reads `ENVIRONMENT` from `Settings`.
- `utils.py`: pure helpers — `.env` file parser, required-setting validator, effective-config aggregator, and template exporter (redacting sensitive keys like `NEO4J_PASS`, `OPENAI_API_KEY`, `API_KEY`).
- `cli.py`: an `argparse` subcommand entry point (`validate`, `summary`, `export`, `flags`, `env`, `provider`, `test-connection`). It prepends the repo root to `sys.path` so it can be invoked as `python -m packages.config.cli <cmd>` from anywhere, and some commands (`test-connection`) reach into sibling packages (`packages.database.graph.graph.Neo4jClient`) to perform live checks.

Dependency direction: `cli.py` → `utils.py` + `feature_flags.py`; `utils.py` imports `settings.py` and `feature_flags.py` only inside functions to avoid circular init-time coupling; `feature_flags.py` imports `settings.py` lazily inside `get_feature_flag_manager`. The package has no internal sub-packages — everything lives at the top level.
