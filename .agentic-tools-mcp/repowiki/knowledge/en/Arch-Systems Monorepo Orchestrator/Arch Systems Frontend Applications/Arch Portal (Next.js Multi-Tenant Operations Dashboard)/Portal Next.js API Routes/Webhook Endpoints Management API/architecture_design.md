Three Next.js App Router route handlers under `/api/webhooks/`: `route.ts` exposes GET (list) and POST (create) against `webhook_endpoints`; `[id]/route.ts` exposes PUT (update) and DELETE (soft-delete via `deleted_at`) for a single endpoint; `[id]/logs/route.ts` reads the last 50 entries from `webhook_delivery_logs`. Each handler follows the same shape: wrap the exported HTTP function with `withRateLimit`, apply `applyCors`, optionally `withBodyLimit` (POST), authenticate via `createServerSupabaseClient().auth.getUser()`, resolve the caller's `employees` row (`department_id`, `role`, `accessible_departments`), enforce admin-or-department ownership before any mutation, then call Supabase directly — no service layer. Validation is delegated to Zod schemas imported from `@/lib/api/schemas` through `validateBody`. After mutations, both list and detail routes revalidate `/admin/tools` and `/[department]/tools` so the UI stays in sync.
