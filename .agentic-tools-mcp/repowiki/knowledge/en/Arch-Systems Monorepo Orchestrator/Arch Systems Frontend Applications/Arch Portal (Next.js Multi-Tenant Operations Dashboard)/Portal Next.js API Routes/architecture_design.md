Each subdirectory under `apps/portal/app/api` maps one-to-one to a URL path via Next.js Route Handlers (`route.ts`), with optional `[id]` or `[table]` dynamic segments for resource-scoped routes (e.g. `/api/webhooks/[id]/logs`, `/api/admin/data/[table]`). Cross-cutting concerns — authentication, RBAC checks against Supabase roles, and shared error/response shapes — are applied uniformly at the top of each handler rather than in a central middleware layer; child summaries confirm this pattern across ai, export, webhooks, and ops_health. Test files co-located alongside their handlers (`*.test.ts`) drive the same route entry points, keeping integration tests colocated with implementation.
