The app is organised by Next.js App Router route groups (`(auth)`, `(hub)`, `(departments)/[department]`, `admin`) plus an `api/` directory of route handlers. All route groups share a top-level `layout.tsx` / `ClientProviders.tsx` shell that wires React Context providers (focus mode, split-window, navigation state), global error/not-found pages, and the system tray. Feature logic is split into two parallel trees: `features/*` holds feature-sliced client components consumed by routes, while `lib/*` owns server-side services (env, cache, audit, department context, shift closeout, weather, monitoring, AI agent, plugin loader, background jobs) invoked from both route handlers and Server Components. Cross-cutting UI lives in `components/` and `hooks/`. A Rust native engine under `plugins/rust-telemetry-engine/` is exposed as a Next.js plugin widget via its `index.tsx` entry. Authentication is enforced through a single `middleware.ts` that gates `(auth)` vs authenticated groups, and every route handler reads identity from the same session cookie set by the login route.
