Single-page client component (`SatelliteMonitoringDashboard.tsx`) owns tab state (`overview | sar | hyperspectral | highres | lidar | raster | kepler`) and renders a left `MonitoringMap` plus a right contextual panel whose content switches on the active tab. Heavy third-party map libraries (`KeplerGlMap`, `LidarLayer`, `COGRasterLayer`, `MonitoringMap`) are imported via Next.js `dynamic()` with `ssr: false` and a spinner placeholder to avoid server-side bundle bloat. Each tab's panel is a self-contained sibling component (`SARLayerPanel`, `HyperspectralLayer`, `HighResPanel`, `DeformationAlertCard`/`DeformationSummary`, `DeformationVelocityChart`). Data comes from `@/lib/monitoring-api` — deformation readings are generated client-side at render time via `generateDeformationReadings(DEFAULT_MINE_CENTER)`, while STAC scene metadata and helper formatters (`formatSceneDate`, `getSTACQuicklookUrl`, `ALERT_THRESHOLDS`, `getSentinel1RevisitDates`) are consumed as pure props. Dependency direction is strictly inward: this module depends on `@/components/monitoring/*`, `@/lib/monitoring-api`, and `@repo/ui/GlassCard`; nothing here imports back into the parent feature tree.
