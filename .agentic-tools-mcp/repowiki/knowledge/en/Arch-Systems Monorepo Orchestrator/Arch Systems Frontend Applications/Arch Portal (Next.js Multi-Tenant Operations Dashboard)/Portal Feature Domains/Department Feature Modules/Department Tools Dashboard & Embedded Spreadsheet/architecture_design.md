Single-page client component (`ToolsPageClient.tsx`) owns the page state: it polls `/api/tools/status` every 30 s to keep a `ToolStatus[]` list in React state and renders each entry via `ToolCard`. The heavy UniverJS spreadsheet is loaded lazily through Next.js `dynamic()` with `ssr: false` so the server bundle stays small; a pulse placeholder is shown while it loads. `UniverSheet.tsx` is a self-contained leaf that mounts/unmounts a Univer workbook inside a ref'd container and disposes the API on unmount. `ToolCard` handles per-tool UI (status badge, Open link, Expand/Collapse toggle) and conditionally renders an `<iframe>` sandboxed for scripts/same-origin/forms/popups/downloads when the tool is online. Dependency direction is one-way: `ToolsPageClient → ToolCard + UniverSheet`; no shared hooks or context are used across these files.
