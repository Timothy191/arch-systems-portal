Each child module is a sibling segment under `apps/portal/app/(departments)/`: the dynamic `[department]/...` routes plus fixed department groups (`access-control`, `drilling`, `engineering`, `training`). All children share the same Next.js App Router conventions — a root `layout.tsx` wrapping a sidebar/navigation shell, a top-level `page.tsx` acting as the department dashboard, and nested sub-routes for features. The parent orchestrates nothing beyond this flat routing topology; cross-department wiring lives in the portal's outer layout/providers, so departments are isolated by route scope rather than by code sharing.
