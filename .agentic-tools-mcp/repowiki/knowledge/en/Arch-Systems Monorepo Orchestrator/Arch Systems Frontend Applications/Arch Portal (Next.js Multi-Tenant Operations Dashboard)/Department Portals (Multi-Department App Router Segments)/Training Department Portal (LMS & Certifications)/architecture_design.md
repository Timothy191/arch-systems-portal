Built as a Next.js App Router route group under `app/(departments)/training/`. The root `layout.tsx` composes the shared `DepartmentLayout` from `@repo/ui`, resolves the training department via `DEPARTMENTS.find` + `getDepartmentTabs("training")`, injects an `ActiveDepartmentSetter`, and wraps children with `AIAssistantWrapper` — establishing the department chrome for every nested route. Each feature (`courses`, `certifications`, `reports`, `schedules`) is its own route segment with a minimal layout that simply re-renders `children`, inheriting the parent department shell. The `page.tsx` at the root renders a static LMS overview dashboard; `courses/page.tsx` demonstrates client-side filtering over an in-memory `initialCourses` array driven by URL search params (`q`, `category`). Shared UI primitives live in `components/`: `SearchForm` (GET form with hidden param forwarding), `FilterTabs` (URL-param-driven tab links using `URLSearchParams`), and `ExportButton` (a `'use client'` leaf with local loading state). All visual styling uses Tailwind classes plus CSS variables like `--text-heading`, `--accent-blue`; data is entirely mock/stub — no server actions or API calls are wired yet.
