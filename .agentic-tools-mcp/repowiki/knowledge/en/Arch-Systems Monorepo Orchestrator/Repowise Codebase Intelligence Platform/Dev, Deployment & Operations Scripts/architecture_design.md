Flat collection of executable entry-point scripts under `scripts/`, each a self-contained CLI with no shared library — cohesion is achieved by convention rather than code reuse. Two families coexist:

- Bash orchestration (`dev.sh`, `deploy.sh`, `setup-production-environment.sh`, `preflight-checklist.sh`, `monitor-hud.sh`, `shutdown.sh`, `deploy-live-local.sh`, `pentest.sh`, `generate-db-docs.sh`, `clear-port.sh`, `copy-assets.sh`) that resolve `REPO_ROOT` via `$(cd "$(dirname "$0")/.." && pwd)`, then call into sibling compose files at the repo root (`docker-compose.tools.yml`, `docker-compose.monitoring.yml`, `docker-compose.production.yml`) and sub-packages (`packages/database`, `apps/portal`, `packages/supabase`).
- A single Python utility (`ensure_reachability.py`) that edits `.env` / `config.toml` in place with regex-based replacements.

Dependency direction is outward only: scripts depend on external toolchains (Node/pnpm, Docker/docker compose v2, Supabase CLI, tbls, OWASP ZAP, rsync, lsof/ss) but never import each other. Cross-cutting concerns (colorized logging, phase banners, healthcheck polling, terminal detection for kitty/gnome/konsole/alacritty/xfce4/xterm) are duplicated per script rather than factored out, keeping every file runnable standalone.
